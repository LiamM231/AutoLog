# מד״כ — אפליקציית תיעוד רכב (PWA)

## דחוף: אם האתר החי מציג HTML לא מעוצב (כמו טקסט רגיל בלי שום סטייל)

זה אומר ש-`styles.css` לא נטען. הסיבה הכי נפוצה: אותה בעיה שכבר
נתקלתם בה - הועלתה **התיקייה עצמה** ולא **התוכן שבתוכה**. ודאו
ש-`index.html` יושב ממש בשורש מה שהעליתם ל-Vercel, לא בתוך תיקיית
משנה. הכי קל לבדוק: תוך כדי גלישה באתר החי, נסו לגשת ישירות ל-
`https://autolog-sigma.vercel.app/styles.css` - אם מוצג "404 Not
Found", זה מאשר שהקובץ לא הועלה נכון.

## חדש (הגדול מכולם): התחברות אמיתית עם חשבון משתמש + מסד נתונים בענן

עד עכשיו כל הנתונים ישבו רק בדפדפן שלכם. עכשיו יש התחברות אמיתית עם
אימייל (בלי סיסמה - "קסם לינק"), והנתונים נשמרים בענן ומשויכים
לחשבון שלכם. זה גם פותר את מגבלת האחסון שהזכרתי בעדכונים קודמים.

### הגדרה נדרשת - חובה, בלי זה האתר לא יעבוד בכלל

1. **פתחו חשבון חינמי ב-https://supabase.com** וצרו פרויקט חדש
   (בוחרים שם וסיסמה למסד הנתונים - שמרו את הסיסמה בצד, לא תצטרכו
   אותה בקוד אבל כן לניהול עתידי)
2. בפרויקט החדש, היכנסו ל-**SQL Editor** (בתפריט הצד) → **New query**
   → העתיקו לשם את **כל** תוכן הקובץ `supabase-schema.sql` שמצורף
   כאן → **Run**. זה יוצר את הטבלאות והרשאות האבטחה.
3. היכנסו ל-**Settings** → **API**. שם תראו שני ערכים שצריך להעתיק:
   - **Project URL** (נראה כמו `https://xxxxx.supabase.co`)
   - **anon public key** (מחרוזת ארוכה)
4. פתחו את `app.js` בפרויקט שלכם, ובשורות הראשונות תמצאו:
   ```js
   const SUPABASE_URL = 'https://YOUR-PROJECT.supabase.co';
   const SUPABASE_ANON_KEY = 'YOUR-ANON-PUBLIC-KEY';
   ```
   החליפו את שני הערכים באלה שהעתקתם. **המפתח הזה (anon key) בטוח
   לחשוף בצד הלקוח** - זה בדיוק התפקיד שלו, בניגוד למפתח ה-Anthropic
   שחייב להישאר בשרת.
5. עדיין ב-Supabase: **Authentication** → **URL Configuration** →
   הוסיפו את כתובת האתר החי שלכם (`https://autolog-sigma.vercel.app`)
   גם תחת **Site URL** וגם תחת **Redirect URLs**. בלי זה, הקישור
   שיישלח במייל לא יחזיר אתכם נכון לאתר.
6. העלו את כל הקבצים המעודכנים ל-Vercel כרגיל.

### איך זה עובד למשתמש
נכנסים לאתר → מזינים אימייל → לוחצים "שליחת קישור כניסה" → פותחים
את המייל שהתקבל → לוחצים על הקישור → חוזרים אוטומטית לאתר, מחוברים.
בפעם הראשונה זה גם יוצר את החשבון (אין הבדל בין "הרשמה" ל"התחברות" -
זה בדיוק העניין בקסם-לינק). יש כפתור התנתקות (סמל היציאה) בפינת
מסך הבית.

### מגבלה לדעת
המבנה הנוכחי תומך ברכב אחד לכל חשבון (בדיוק כמו שהיה עד עכשיו) -
פשוט עכשיו זה מסונכרן בענן במקום נעול למכשיר אחד. תמיכה בכמה רכבים
לאותו משתמש היא הרחבה טבעית להמשך, אם תרצו.

5. אחרי כ-30 שניות תקבלו כתובת חיה, למשל: `car-care-app.vercel.app`

זהו - האתר חי באינטרנט. אפשר לשלוח את הכתובת הזו לכל אחד.

**חלופה:** אם תרצו בעתיד לחבר דומיין אמיתי (כמו `carcare.co.il`),

## מה יש בפרויקט

- `index.html` - כל שלושת המסכים (בית, הוספת רכב, הוספת טיפול)
- `styles.css` - העיצוב (פלטת גרפיט + ברונזה, בהשראת לוח מכשירים)
- `app.js` - כל הלוגיקה: שמירה מקומית בדפדפן (localStorage), חיפוש
  רכב מול data.gov.il, חישוב וציור ציר הטיפולים
- `manifest.json` - הגדרות ה"התקנה" (שם, אייקון, צבעים)
- `service-worker.js` - טעינה מהירה יותר בביקורים חוזרים
- `icons/` - אייקוני האפליקציה

## חדש: עיבוד אוטומטי של קבלות (AutoLog)

עכשיו כשמצלמים קבלה במסך "הוספת תיעוד", היא נשלחת לניתוח AI אמיתי
(Claude) שקורא את הקבלה ומנסה למלא אוטומטית: תאריך, ק"מ, מה הוחלף,
מחיר ומק"ט. המשתמש עדיין רואה ויכול לתקן את השדות לפני השמירה -
זה מכוון, כדי שתמיד תהיה שליטה על מה שנשמר.

### הגדרה נדרשת - חובה לפני שזה יעבוד

1. היכנסו ל-https://console.anthropic.com וצרו מפתח API (תחת
   Settings -> API Keys). תצטרכו להטעין קרדיט קטן (אפשר להתחיל
   בכמה דולרים בודדים - כל סריקת קבלה עולה שברירי אגורות)
2. ב-Vercel, היכנסו לפרויקט → **Settings** → **Environment Variables**
3. הוסיפו משתנה בשם `ANTHROPIC_API_KEY` עם הערך שקיבלתם משלב 1
4. **חשוב:** אחרי הוספת המשתנה צריך **Redeploy** כדי שהוא ייכנס
   לתוקף (בטאב Deployments, שלוש נקודות ליד הדפלוימנט האחרון →
   Redeploy)

### מבנה טכני חדש בפרויקט
- `api/parse-receipt.js` - פונקציית שרת (Vercel Serverless Function)
  שמקבלת את תמונת הקבלה, שולחת אותה ל-Claude, ומחזירה JSON מובנה.
  מפתח ה-API נשאר בצד השרת ולעולם לא נחשף לדפדפן.

### המלצה: מעכשיו כדאי לעבור לפריסה דרך GitHub

עד עכשיו העלינו קבצים ידנית (גרירה ל-Vercel). מרגע שיש קוד שרת
(`api/`), הרבה יותר נוח ואמין לעבוד עם **Connect Git** (הכפתור
שראיתם למעלה במסך ה-Deployment ב-Vercel):
1. פתחו חשבון חינמי ב-github.com אם אין לכם
2. צרו repository חדש (למשל בשם `autolog-app`)
3. תחת "Add file" → "Upload files" באתר GitHub, גררו את כל תוכן
   התיקייה `car_care_web` (בלי git command line, ישירות מהדפדפן)
4. ב-Vercel, בפרויקט הקיים → Settings → Git → **Connect Git Repository**,
   ובחרו את ה-repo שיצרתם
5. מכאן והלאה, כל שינוי שתעלו ל-GitHub יעדכן את האתר החי אוטומטית תוך שניות

## חדש: כל פיצ'רי ה-AI עכשיו אמיתיים

בעדכון הזה שלושה פיצ'רים עברו מ"שמירה בלבד" ל-AI אמיתי שמפעיל ניתוח:

1. **קבלות** (`api/parse-receipt.js`) - כמו קודם, קורא ומתעד אוטומטית.
2. **ספר רכב** (`api/parse-service-book.js`) - **חדש**. אחרי שמעלים
   תמונות של עמודי ספר הרכב, לוחצים על הכפתור "ניתוח עם AI וחילוץ
   מרווחי טיפול". ה-AI קורא את טבלת התחזוקה של היצרן ומעדכן אוטומטית
   את המרווחים שלפיהם מחושב **ציר הטיפולים** (במקום ברירת המחדל
   הכללית שהייתה עד עכשיו).
3. **יבוא אקסל** (`api/parse-spreadsheet.js`) - **שודרג לגמרי**.
   במקום התאמת מילות מפתח קבועה (שדרשה כותרות עמודות מדויקות),
   עכשיו זה שולח את הקובץ כמו שהוא ל-AI, שמבין את המבנה בעצמו -
   גם אם העמודות בסדר לא צפוי, בעברית חופשית, או בלי כותרות ברורות
   בכלל.

כל שלושת אלה משתמשים באותו `ANTHROPIC_API_KEY` שכבר הגדרתם. אין
צורך בהגדרה נוספת - אם עיבוד הקבלות כבר עבד, השניים האחרים יעבדו
גם הם ברגע שתעדכנו את הקבצים ב-Vercel.

**לגבי עלות:** ניתוח ספר רכב ויבוא אקסל משתמשים במודל חזק יותר
(Sonnet) כי המשימה מורכבת יותר מקריאת קבלה בודדת - עדיין מדובר
בעלות זניחה (סנטים בודדים גם בשימוש מרובה), אבל שווה לדעת שזה קצת
יותר יקר מסריקת קבלה רגילה.

**מגבלה לדעת:** יבוא אקסל מוגבל לכ-20,000 תווים מהקובץ (בערך כמה
מאות שורות) בבקשה אחת, כדי לא לחרוג ממגבלת האורך של הבקשה ל-AI.
לקובץ ענק במיוחד - כדאי לפצל לכמה קבצים קטנים יותר ולייבא בנפרד.

## חדש: ייצוא ספר טיפולים + פרטי חלק והזמנה

1. **ייצוא ספר טיפולים** - כפתור בתחתית מסך הבית, מייצר קובץ Excel
   מסודר עם שני גיליונות: פרטי הרכב, וטבלת כל הטיפולים ממוינת
   לפי ק"מ (מהישן לחדש) עם סה"כ הוצאות. מיועד בדיוק למצב של מכירת
   הרכב - קונה פוטנציאלי יכול לראות תיעוד מסודר. זה רץ כולו בדפדפן,
   בלי AI ובלי שרת בכלל.
2. **לחיצה על אירוע בציר הטיפולים** - פותחת כרטיס עם שם החלק,
   מתי בוצע/צפוי, וקישורי חיפוש אמיתיים (eBay Motors, AliExpress,
   Google Shopping) עם יצרן+דגם+שנה+שם החלק כבר מוזנים בחיפוש.

   **החלטת עיצוב מכוונת:** בחרתי **לא** לתת ל-AI "להמציא" מספרי
   מק"ט - זה מסוכן מדי לחלקי בטיחות (בלמים, רצועת טיימינג וכו').
   AI יכול "להיות בטוח" במספר שגוי, ואי אפשר לדעת מתי. במקום זה,
   הקישורים בנויים בצורה קבועה (בלי AI, בלי עלות) עם חיפוש מדויק
   לפי פרטי הרכב, כדי שהמשתמש יראה בעצמו תמונות ומק"טים אמיתיים
   באתרים, ולוודא מול המוסך לפני הזמנה בפועל.

## עדכון: קישורי החלקים הוחלפו ל-PartSouq ו-Cars245

  אז במקום לתת קישור שעלול להיות שבור, הקישור מפנה לדף הבית שלהם
  ומשם בוחרים את הרכב ידנית. אם תמצאו אצלם כתובת חיפוש שעובדת
  בפועל, אפשר לעדכן בקלות ב-`app.js` בפונקציה `showPartDetail`.

## עדכון: קישור עזר ל-Govcar לאיתור VIN

בדקתי את govcar.co.il - זה כלי צד-שלישי לא רשמי (מצהיר בעצמו שהמידע
"ממקורות שונים וגלויים" ושאין אחריות לדיוק), בלי API או כתובת חיפוש
עם פרמטר טקסט שאפשר לבנות באופן אמין. **לא שילבתי scraping** אוטומטי
מולו - זה שביר (המבנה שלהם יכול להשתנות בכל רגע) ובעייתי מבחינת
תנאי שימוש. במקום זה יש קישור נוח ליד שדה ה-VIN שפותח את govcar
בלשונית חדשה, כדי שתבדקו שם ידנית ותדביקו את המספר אצלנו.

## עדכון: חיפוש מדויק ב-PartSouq לפי מספר שלדה (VIN)

בדקתי - מאגר `data.gov.il` הפתוח (שדרכו מזהים רכב לפי מספר רישוי)
כנראה **לא כולל** את מספר השלדה, גם שהוא קיים במאגר הפנימי המלא של
משרד התחבורה (כנראה משיקולי פרטיות). אז אי אפשר למשוך אותו אוטומטית.

**הפתרון:** ליד שם הרכב במסך הבית יש עכשיו שורה קטנה "+ הוספת מספר
שלדה (VIN)" - לוחצים, מזינים את המספר (מודפס על רישיון הרכב, גם
מוטבע פיזית על הרכב), ונשמר. מרגע שיש VIN שמור, כרטיס פרטי החלק
(שנפתח בלחיצה על הציר) ישתמש אוטומטית בחיפוש PartSouq לפי VIN
(`partsouq.com/search/vin?vin=...`) שהוא מדויק משמעותית מחיפוש טקסט
חופשי. אם אין VIN שמור, זה חוזר לחיפוש טקסט רגיל.

## עדכון: מגע עיצוב סופי - חמימות, תנועה וסאונד

1. **אנימציית מחוג בפתיחה** - הלוגו של AutoLog עכשיו "מתעורר לחיים":
   המחוג סוקר מאפס למקומו הסופי, עם זוהר עדין סביב האייקון - בדיוק
   כמו הרגע שמדליקים רכב. חד-פעמי, לא חוזר על עצמו, ומכבד את הגדרת
   `prefers-reduced-motion` (מתבטל אוטומטית למי שהגדיר במכשיר שלו
   פחות אנימציות).
2. **צליל מנוע בכניסה** - סינתטי לגמרי (Web Audio API), לא קובץ
   MP3 חיצוני - כך שזה תמיד עובד בלי תלות בזמינות שרת אחר. בדפדפנים
   שחוסמים סאונד אוטומטי בטעינה (מדיניות סטנדרטית), הוא יתנגן
   בנגיעה הראשונה במסך במקום.
3. **חימום קטן בטון** - טקסט מסך "אין רכבים" נהיה יותר מזמין.

## מה נוסף בעדכון הקודם

1. **תמונת רכב עם הסרת רקע אוטומטית** - לוחצים על תיבת התמונה בראש
   מסך הבית, בוחרים תמונה, והרקע מוסר אוטומטית **ישירות בדפדפן**
   (אין שרת מעורב - הכל רץ עם ONNX/WebAssembly על המכשיר עצמו).
   בפעם הראשונה זה מוריד מודל AI (כ-40MB) ולכן עלול לקחת כמה שניות
   או יותר בחיבור איטי - בפעמים הבאות זה מהיר יותר כי המודל שמור
   בקאש של הדפדפן.
2. **עמודי ספר רכב** - אפשר להעלות כמה תמונות של עמודי ספר הרכב.
   כרגע הן רק נשמרות לצפייה - **לא** מבוצע עליהן חילוץ אוטומטי של
   מרווחי הטיפול. זה דורש שירות AI עם שרת (ראו הסבר למטה).
3. **עריכת ק"מ נוכחי** - לחיצה על תיבת "ק״מ נוכחי" בעמוד הבית פותחת
   חלון עריכה מהיר.
4. **צילום קבלה** - במסך "הוספת תיעוד" אפשר לצרף תמונה של הקבלה
   לכל רשומה.
5. **יבוא מאקסל** - כפתור "יבוא מאקסל" ליד היסטוריית הטיפולים. הקובץ
   צריך שורת כותרות ראשונה עם עמודות שהמערכת מזהה אוטומטית:
   `תאריך`, `ק"מ`, `מה הוחלף`, `מחיר`, `בוצע` (מוסך/עצמי), `מק"ט`.
   סדר העמודות לא משנה, ואפשר גם באנגלית (date, km, part, price...).

## מגבלת אחסון חשובה - קראו לפני שמעלים הרבה תמונות

כל הנתונים (כולל התמונות!) עדיין נשמרים רק ב-localStorage של
הדפדפן - וזה מוגבל בדרך כלל ל-5-10MB. דחסתי את התמונות אוטומטית
כדי לצמצם את הבעיה, אבל אם תעלו הרבה מאוד קבלות/עמודי ספר רכב,
בסופו של דבר זה יתמלא ותקבלו שגיאה בשמירה. זה בדיוק המקום שבו
המעבר לאחסון ענן (Firebase Storage / Supabase Storage) הופך
להיות חשוב - זה הצעד המומלץ הבא.

## איך ממשיכים לשלב הבא: חילוץ אוטומטי מספר הרכב (OCR/AI אמיתי)

כדי שהמערכת "תדע לקרוא" את מרווחי הטיפול מתוך תמונת ספר הרכב בפועל,
צריך לשלוח את התמונה לשירות בינה מלאכותית עם יכולת ראייה (כמו
Claude API עם תמיכה בתמונות, או Google Cloud Vision), ולקבל בחזרה
טקסט/נתונים מובנים. זה דורש **שרת קטן** (backend) שמחזיק את מפתח
ה-API בצורה מאובטחת - אי אפשר לשים מפתח API ישירות בקוד שרץ בדפדפן
של המשתמש, כי כל אחד יוכל לראות ולגנוב אותו. זה השלב הטבעי הבא
בפיתוח, אחרי שהגרסה הנוכחית תוכיח את עצמה.

## מגבלות השלב הנוכחי (חשוב לדעת)

1. **הנתונים נשמרים רק בדפדפן הזה, במכשיר הזה.** אם תפתחו את האתר
   ממכשיר אחר, או תנקו את נתוני הדפדפן, המידע ייעלם. זה מספיק כדי
   לבדוק את הרעיון, אבל לפני שמשתמשים אמיתיים סומכים על זה - צריך
   להוסיף התחברות + מסד נתונים בענן (Firebase/Supabase, כמו שדיברנו).
2. **resource_id של data.gov.il** בקובץ `app.js` כדאי לוודא מול
   https://data.gov.il/dataset/private-and-commercial-vehicles
   שהוא עדיין תקף.
3. **OCR לצילום קבלות/ספר רכב** עדיין לא ממומש - זה הצעד הבא.
4. **התראות Push** לא ממומשות עדיין - אפשר להוסיף בהמשך, עם מגבלת
   iOS ל-16.4 ומעלה.
